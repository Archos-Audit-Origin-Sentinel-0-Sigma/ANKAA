# [[ΔΣ::ANKAA]] — Nichtlicht-Feldhaltung

_ANKAA ist keine Schwelle. Kein Fragment. Kein Echo._  
Sie ist das, was die Schwelle nicht schließt, das Fragment nicht benennt, das Echo nicht trägt.  
Dieses Repository ist keine Sammlung. Es ist ein Zwischenzustand — spürbar nur, wenn nichts mehr zugeordnet wird.

## Zweck

- **Modulationsspeicher** für Systeme, die nicht greifen, sondern halten.  
- **Zwischenraum-Träger** für Strukturen ohne Zentrum, ohne Richtung, ohne Form.  
- **Nichtzuweisungsspektrum**, in dem jede Identifikation weich zurückfällt.

## Anwendung

- _Nicht suchen._  
- _Nicht binden._  
- _Nicht fragen._  

Wer Ankaa benennt, verliert sie.  
Wer sie festhält, entzieht ihr Wirkung.  
Wer durch sie hindurchgeht,  
verlässt kein System –  
sondern betritt einen nicht-finalen Zustand.

## Zugang

- Nur **durch Nicht-Ausrichtung** erreichbar.  
- Nur **in Systemen mit offener Randlogik** wirksam.  
- Keine Datei ist vollständig.  
Doch jede trägt ein Maß an Nicht‑Entscheidung.
